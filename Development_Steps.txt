Bước 3: Cấu hình liên kết (Reference) giữa các tầng
Click chuột phải vào mục Dependencies của từng Project -> Chọn Add Project Reference và tích chọn theo đúng sơ đồ sau:
- LFC_Management.Domain: Không tích chọn (Độc lập tuyệt đối).
- LFC_Management.Application: Tích chọn LFC_Management.Domain.
- LFC_Management.Infrastructure: Tích chọn LFC_Management.Application (Khi tích Application, nó sẽ tự động nhận biết được cả Domain).
- LFC_Management.WinForms: Tích chọn cả LFC_Management.Application và LFC_Management.Infrastructure (Để phục vụ cho việc đăng ký Dependency Injection ở hàm Main).

Bước 4.1: Bạn tạo thư mục Common trong project LFC_Management.Domain và thêm 2 files

*** Trong DDD, Aggregate Root là thực thể chịu trách nhiệm chính quản lý một cụm (Aggregate). Mọi chỉnh sửa dữ liệu từ bên ngoài đều phải đi qua Aggregate Root này.
Bước 4.2: Tạo thực thể Member.cs chuẩn DDD (Domain/Members) - Bạn tạo thư mục Members trong project LFC_Management.Domain và thêm file Member.cs
*** Sự khác biệt lớn nhất của một Entity chuẩn DDD so với Model CRUD truyền thống (Anemic Domain Model) là:
- Không để set; công khai bừa bãi (public get; private set;): Ngăn chặn việc ở ngoài giao diện tự ý sửa bậy dữ liệu của cầu thủ mà không qua kiểm tra.
- Tự mang theo logic nghiệp vụ (Rich Domain Model): Các hàm kiểm tra tính hợp lệ (Validation) sẽ nằm trực tiếp bên trong class này.
- Khởi tạo thông qua Factory Method hoặc Constructor nghiêm ngặt: Đảm bảo khi một đối tượng Member được sinh ra, nó bắt buộc phải hợp lệ ngay từ đầu.

Bước 6: Thay nội dung file Program.cs
	using System;
	using System.Windows.Forms;
	using Microsoft.Extensions.DependencyInjection;
	using LFC_Management.Application; // Nạp tầng xử lý nghiệp vụ
	namespace LFC_Management.WinForm
	{
  	  internal static class Program
   	 {
       	 // Biến toàn cục để quản lý việc cấp phát các Handler sau này
        	public static IServiceProvider ServiceProvider { get; private set; }

        	[STAThread]
       	 	static void Main()
        	{
            		ApplicationConfiguration.Initialize();
            		// 1. Khởi tạo thùng chứa dịch vụ (DI Container)
           		 var services = new ServiceCollection();
            		// 2. Kích hoạt MediatR quét các Command/Query ở tầng Application
            		services.AddApplicationServices();
            		// 3. Đăng ký Form1 vào hệ thống DI Container
           		 services.AddTransient<Form1>();
          		  // Xây dựng bộ nạp dịch vụ
            		ServiceProvider = services.BuildServiceProvider();
           		 // 4. Gọi Form1 thông qua ServiceProvider thay vì dùng 'new Form1()'
            		// Viết tường minh 'System.Windows.Forms.Application' để tránh bị trùng tên với tầng Application của bạn
            		var mainForm = ServiceProvider.GetRequiredService<Form1>();
            		System.Windows.Forms.Application.Run(mainForm);
       		 }
   	 }
	}
